# Style Guide

This document outlines the writing and formatting standards for the EPIC RPG rules and setting materials.

## Terminology

### Character vs. Hero

**Always use "Character" instead of "Hero".**

- ✅ Correct: "A Character receives talent points..."
- ✅ Correct: "Characters of any Grade receive..."
- ❌ Incorrect: "A Hero receives talent points..."
- ❌ Incorrect: "Heroes of any Grade receive..."

This applies to all forms:
- Character (singular)
- Characters (plural)
- Character's (possessive)
- Characters' (plural possessive)

### Characteristic Points vs. Improvement Points

**Always use "Characteristic Points" instead of "Improvement Points".**

- ✅ Correct: "Characteristic points at character creation..."
- ✅ Correct: "How many characteristic points can be allocated"
- ❌ Incorrect: "Improvement points at character creation..."
- ❌ Incorrect: "How many improvement points can be allocated"

### Skills vs. Special Skills

**Always use "Skills" instead of "Special Skills".**

- ✅ Correct: "A Character's Skills..."
- ✅ Correct: "skill points"
- ❌ Incorrect: "A Character's Special Skills..."
- ❌ Incorrect: "special skill points"

### Armor Spelling

**Always spell "armor" with no "u" (American spelling).**

- ✅ Correct: "armor", "armored", "armor-piercing"
- ❌ Incorrect: "armour", "armoured", "armour-piercing"

This applies to all forms and contexts.

## Capitalization

- **Characteristics**: PHYSICAL, MENTAL, LUCK (always uppercase)
- **Derived values**: STAMINA POINTS, PSI POINTS, LUCK POINTS (always uppercase)
- **Skills**: Capitalize when referring to the skill name (e.g., "the Firearms skill"), lowercase when referring to the concept (e.g., "skill points")
- **Talents**: Capitalize when referring to a specific talent name (e.g., "the Packhorse Talent"), lowercase when referring to the concept (e.g., "talent points")

## Game Master vs. Director

**Use "Game Master" for system-neutral terminology.**

- ✅ Correct: "at the Game Master's discretion"
- ❌ Incorrect: "at the Director's discretion"

## Formatting

- Use markdown formatting consistently
- Tables should be properly formatted with aligned columns
- Code references should use the format: `startLine:endLine:filepath`
- Links should use wiki-style links: `[[RULES/File-Name|Display Text]]`
